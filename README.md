# Projet Fil Rouge — Ventes & Stock de Pièces Automobiles

## Contexte
Dans le cadre de la formation Data Analyst , ce projet fil rouge vise à concevoir une solution data décisionnelle complète, couvrant l’ensemble du cycle de vie de la donnée, de la collecte à la restitution pour la prise de décision.

Le projet s’inscrit dans un contexte métier réel : la vente de pièces automobiles, un secteur confronté à des problématiques de gestion de stock, de performance commerciale et d’optimisation des décisions.

---

## Problématique
Comment exploiter les données de ventes et de stock de pièces automobiles afin d’optimiser la gestion des stocks, réduire les ruptures et améliorer la performance commerciale ?

---

## Objectifs du projet
- Analyser les ventes par pièce, catégorie et période
- Identifier les pièces à forte et faible rotation
- Détecter les risques de rupture de stock
- Aider à la prise de décision de réapprovisionnement

---

## Sources de données
- Données de ventes (CSV)
- Données de stock (CSV)
- Données produits / pièces automobiles (CSV)
- Données fournisseurs (CSV)
- Données externes via API (météo ou prix du carburant)

---

## Architecture & pipeline data
1. Extraction des données (CSV + API)
2. Nettoyage et préparation des données (Python / pandas)
3. Stockage en base de données relationnelle (PostgreSQL)
4. Analyse des données et calcul des KPI (SQL / Python)
5. Visualisation et restitution décisionnelle (Power BI)
